# 🚀 백엔드 A1 인스턴스 마이그레이션 결과 보고서

**작성일자:** 2026년 06월 16일  
**작성자:** Antigravity AI Agent  
**프로젝트:** PropAI Platform Backend

---

## 1. 개요 (Overview)
기존 x86 아키텍처 기반의 사양 제한적인 OCI Micro 인스턴스에서 발생하는 자원 부족(OOM 및 지연) 문제를 해결하기 위해, OCI Always Free 티어의 최대 한도를 활용한 **고성능 ARM(Aarch64) 기반 A1 인스턴스**로 백엔드 서버를 무중단 마이그레이션하는 작업을 수행했습니다.

---

## 2. 인프라 변경 사항 (Infrastructure Changes)

| 구분 | 기존 환경 (Old) | 신규 환경 (New) | 비고 |
| :--- | :--- | :--- | :--- |
| **IP 주소** | `134.185.104.167` | **`168.110.125.89`** | Cloudflare DNS 전환 완료 |
| **인스턴스 타입** | `VM.Standard.E2.1.Micro` | **`VM.Standard.A1.Flex`** | - |
| **아키텍처** | x86_64 (AMD64) | **aarch64 (ARM64)** | AI/BIM 의존성 호환성 확보 |
| **CPU (OCPU)** | 1 OCPU | **3 OCPU** | 연산 성능 3배 증가 |
| **Memory (RAM)**| 1 GB | **18 GB** | 메모리 18배 증가 (OOM 완전 해소) |
| **데이터베이스** | Supabase (외부 클라우드) | Supabase (외부 클라우드) | DB 무중단 유지 |

> **💡 OCI 무료 자원 최적화:**  
> 백엔드(3 OCPU/18GB) + 프론트엔드(1 OCPU/6GB) = 총 4 OCPU / 24GB RAM으로 오라클 클라우드 평생 무료(Always Free) 최대 한도를 100% 최적화하여 구성했습니다.

---

## 3. 단계별 마이그레이션 수행 과정 (Migration Process)

### [Phase A] 신규 A1 인스턴스 프로비저닝 (완료)
- OCI 콘솔을 통해 3 OCPU, 18GB RAM 사양의 신규 A1 인스턴스 생성
- SSH 키 교환 및 OCI 방화벽(Security List) 설정 검증

### [Phase B] 애플리케이션 및 환경 이관 (완료)
1. **환경 변수 및 설정 파일 이관**
   - 구형 서버에 SSH 접속하여 `propai-platform` 디렉토리 내 운영 환경변수(`.env`), 리버스 프록시 설정(`~/caddy/Caddyfile`), 무중단 배포 스크립트(`deploy.sh`) 백업 및 신규 서버로 전송.
2. **ARM 네이티브 환경 재구축**
   - `docker`, `docker-compose-plugin`, `git` 등 필수 패키지 설치.
   - 로컬 네트워크 통신용 Redis 및 Qdrant 벡터 DB 설치 후 백그라운드 기동 완료.
3. **앱 컨테이너 빌드 및 배포**
   - 기존 x86 이미지가 아닌 ARM 아키텍처에서 동작하도록 구성된 `Dockerfile.oracle`을 사용하여 네이티브 빌드(Compile) 수행.
   - 무거운 파이썬 AI/BIM 패키지(`ifcopenshell`, `xgboost`, `langchain` 등) 설치 호환성 문제 완벽 해결.
4. **Health Check 및 무중단 스위칭**
   - 기존의 `deploy.sh` 로직을 그대로 활용하여 `8001`번 포트로 신규 컨테이너 기동.
   - 내부 `/health` 엔드포인트 응답 검증(HTTP 200 OK) 후 Caddy 프록시를 통해 무중단 스위칭 완료.

### [Phase C] 컷오버 및 트래픽 전환 (완료)
1. **Cloudflare DNS A 레코드 변경**
   - `api.4t8t.net`의 대상 IP를 `168.110.125.89`로 변경 적용 완료 (프록시 활성화 유지).
2. **보안/인증 IP 허용 처리**
   - 카카오 디벨로퍼스 설정의 `REST API 키` 상세 옵션에서 **[호출 허용 IP 주소]** 항목에 신규 서버 IP 추가 완료.

## 4. 기대 효과 (Impact & Results)
- **OOM(Out Of Memory) 장애 제로화:** 기존 1GB 메모리 환경에서 모델을 불러오거나 3D 도면 파싱 시 발생하던 애플리케이션 크래시 현상을 18GB의 넉넉한 메모리로 원천 차단했습니다.
- **AI 응답 속도 및 동시 처리량 극대화:** 3 OCPU의 풍부한 연산 자원을 통해 딥러닝 기반 기능의 응답성이 비약적으로 상승했습니다.
- **비용 최적화:** OCI 한도를 꽉 채워 설계함으로써 인프라 성능은 극한으로 끌어올리면서 추가적인 클라우드 과금은 발생하지 않도록 조치했습니다.

## 5. 향후 권장 사항 (Next Steps)
- 신규 서버로 트래픽이 안정적으로 유입되는 것을 약 **1~2일간 모니터링** 합니다.
- 모든 API 기능이 에러 없이 동작하는 것을 확인한 후, 기존 **Micro 인스턴스(134.185.104.167)는 OCI 콘솔에서 Terminate(종료/삭제)** 처리하여 불필요한 관리를 줄이는 것을 권장합니다.
