# CAD/BIM × LLM 엔지니어링 협업 시스템 — 구현 계획

작성 2026-06. 목표: CAD/BIM 전반에 LLM을 **할루시네이션 없이** 최적 연동하여,
엔지니어링 프롬프트로 **정밀·정확한 도면/CAD/BIM 생성·수정·추가·편집**과
**사용자–LLM 다층·다각 협업**을 가능하게 한다.

## 0. 핵심 원칙 (할루시네이션 구조적 차단)

> **LLM은 좌표·치수·수치를 직접 만들지 않는다.**
> 결정론적 기하 커널(AutoDesignEngine)이 **유일한 진실원(SSOT)** 이며,
> 화면에 표시되는 모든 숫자(면적·세대수·건폐율·용적률·주차)는 커널 산출값이다.

LLM의 역할은 **"검증된 설계 도구(tool/function)를 호출하는 운영자"**:
1. 사용자 자연어 → LLM이 **구조화된 도구호출**(set_unit_mix, set_floors, move_core …) 생성
2. 모든 도구호출은 **법규·공학 제약 검증(ConstraintValidator)** 통과 후에만 적용
3. 위반 시 거부 + 사유를 LLM에 반환 → LLM 재시도 → 합법 스펙만 커널에 투입
4. 생성 후 **검증·근거(grounding) 게이트**가 커널 실제값과 LLM 주장 대조

→ LLM이 만들 수 있는 건 "검증되는 변경요청"뿐이라 가짜 수치가 화면에 도달 불가.

## 1. 5계층 아키텍처

| 계층 | 역할 | 구현(기존→확장) |
|---|---|---|
| L1 진실원(기하 커널) | 파라미터→기하·면적·법규한도 **계산** | AutoDesignEngine(있음) + **DesignSpec**(신규 검증 스펙) |
| L2 제약 검증 | 건폐율·용적률·높이·복도·이격·주차·일조 하드가드 | **ConstraintValidator**(신규) |
| L3 LLM 도구운영 | 자연어→검증되는 tool-call(function calling) | **DesignToolOperator**(신규, BaseInterpreter 토큰계측) |
| L4 검증·협업 | grounding 게이트 + 다각 렌즈 + 대화형 편집 | expert_panel_graph 패턴 + **DesignChatGraph**(신규) |
| L5 BIM 전파 | 스펙→IFC→glTF→QTO 단일기하 전파 | IfcGeneratorService(있음)/ /design/{id}/mass(있음) |

## 2. 할루시네이션 방지 메커니즘 (L2 핵심)

- **스펙 검증**: 복도폭 ≥ 용도별 최소, 이격 ≥ 법정, 층수×층고 ≤ 최고높이, 용도·평형 유효성.
- **기하 검증**: 커널 산출 건폐율 ≤ 법정, 용적률 ≤ 법정, 주차대수 ≥ 법정, 세대면적 합 ≤ 연면적.
- **근거 게이트**: LLM 설명문의 모든 수치를 커널 산출값과 대조, 불일치는 flagged(표시 배제).
- **정직 표기**: 데이터 없는 항목은 "근거 없음" 명시(가짜 수치 금지) — 기존 GROUNDING_RULE 계승.

## 3. 사용자–LLM 다층·다각 협업 모델 (L4)

- **다층(turn loop)**: 사용자 요청 → LLM 도구호출+근거 → 미리보기 → 사용자 승인/수정/거부 → 적용 → LLM 결과설명+다음 제안. design_versions에 매 턴 영속(되돌리기).
- **다각(lens panel)**: 수익성·거주성·법규·시공성 4렌즈 병렬 제안 → 종합(Top-N 대안, 기존 design-alternatives 확장).
- **편집/추가**: 벽 이동·세대 추가·코어 변경도 동일 검증 도구 API 경유 → 2D·3D·BIM 동시 갱신(단일기하).

## 4. 순차 구현 로드맵

- **P1 (토대)**: DesignSpec(검증 스펙) + ConstraintValidator(제약 하드가드). ★할루시네이션 방지 핵심. (이번 단계)
- **P2**: DesignToolOperator — function-calling 도구 API + 검증·재시도 루프(자연어→합법 스펙 편집).
- **P3**: 근거(grounding) 검증 게이트 — 커널값 대조, 가짜수치 차단(verify 노드).
- **P4**: 대화형 협업 UI — 멀티턴 편집 루프(미리보기·승인·되돌리기), design_versions 연동.
- **P5**: 다각 렌즈 패널 + 대안 종합(수익/거주/법규/시공).
- **P6**: LLM 보조 CAD/BIM 편집(벽·세대·코어) → IFC/glTF/QTO 단일기하 전파.
- **P7(선택)**: 표준설계 참조 라이브러리(관리자 업로드 DXF/PDF → 유사평면 검색 → 생성 컨텍스트 주입).

각 단계: 구현 → 컴파일/임포트 검증 → 배포(Micro/A1) → 라이브 검증 → 보고. 무목업·정직 원칙 유지.

## 4-1. 경쟁 플랫폼 벤치마킹 결론 (조사 반영)

| 플랫폼 | 입력 | LLM/NL | 할루시네이션 대응 | 시사점 |
|---|---|---|---|---|
| **Hypar** | 텍스트→BIM | NL=의도파싱만, 기하=결정론 함수 | 원천 차단(생성은 함수) | ★우리 L1/L2와 동일 = 정답 |
| TestFit/Archistar | 파라미터 | 미사용 | 규칙기반 결정론 | 편집가능 1차안 패턴 |
| Architechtures | 파라미터 | 미사용 | 제약 내 생성("error-free") | 스마트 디폴트 필요성 |
| Maket.ai | 자연어+파라미터 | 의도파싱 | 생성후 조닝 대조 | 비전문가 라이트 UX |
| Forma(Spacemaker) | 스케치 | 보조 Assistant | 물리 시뮬 검증 | 실시간 정량 피드백 |
| Finch | 파라미터+그래프 | 제약솔버 | Graph Rules 내장 | 변경 전파 가시화 |
| Digital Blue Foam | 자연어 질의 | GPT KPI/SWOT | 결정론 분석 후행 | NL 인터페이스 최적용 |
| Swapp.ai | 스키매틱 | 룰셋 학습 | "generative 거부" | 신뢰=검증동행 |

**결론**: 업계 정답 = **생성 유창성(LLM NL) + 결정론 검증(엔진)** 하이브리드. 순수 LLM 생성은 환각 불가피.
→ 우리 L1(커널)/L2(검증) 아키텍처가 Hypar 패턴과 일치 = 방향 검증됨.

## 4-2. 비전문가 UX · 음성 인터페이스 (리팩토링 핵심)

**비전문가가 퀄리티 설계를 얻는 6패턴**(전 플랫폼 공통):
1. 프리셋/타이폴로지 스마트 디폴트(빈 캔버스 금지) 2. 실시간 정량 피드백 루프
3. 가이드레일(제약 입력단계 강제) 4. 스타일/예시 변환("모던") 5. 편집가능 1차안+점진정제
6. 할루시네이션 배지(신뢰 가시화 — 기존 VerificationBadge 계승)

**음성 인터페이스(도메인 공백 = 선점 기회)**: STT(ASR) → Claude NLU(의도·엔티티) → **검증 도구호출** → 결정론 엔진.
음성은 "입력 단순화"만 담당, 산출은 기존 결정론 파이프라인 재사용 → 환각 위험 0.
구현: 브라우저 Web Speech API(STT) 우선(무비용) → 정확도 필요 시 서버 STT. 음성/텍스트 동일 의도파서 공유.

## 4-3. 우리만의 차별화 해자 (경쟁사 전무 영역)
1. **한국 법규 실시간 검증형 생성** — 조례 계층(상위법령→도시계획→조례→인센티브)까지(기존 auto_zoning/regulation 결합)
2. **도급현실 수지 검산** — 매출=추정가×낙찰가율, 실원가율0.87(경쟁사 takeoff는 단순물량×단가)
3. **음성 우선 비전문가 UX** — 설계 도메인 최초
4. **청약/분양 데이터 역방향 실현검증** — 유닛믹스→실제 분양실현성(v62 ERP 결합)
5. **전주기 모세혈관 + 검증에이전트 전수검증** — generative + 전수검증 동행

## 4-4. 리팩토링 방침(기존 자산 보존·확장)
- 기존 `AutoDesignEngine`·`DesignIntentInterpreter`·`design-alternatives`·`design_versions`·`IfcGenerator` **보존**.
- 신규 L1/L2(DesignSpec·Validator)를 **그 사이에 끼워** 의도파싱→스펙→검증→커널→검증 흐름으로 재배선.
- 자연어/음성은 동일 의도파서 공유. 화면 수치는 전부 커널값(LLM 미생성).

## 5. 참조자료(P7) 현실
공개 표준설계(LH 표준주택형·행복주택 표준설계도 등)는 대부분 PDF/도서 형태 → 메타+비전 추출.
대량 CAD 데이터셋은 제한적 → **관리자 업로드가 핵심**. DXF=ezdxf 파싱(설치됨), DWG=변환 필요, PDF/이미지=비전. 저작권 확인 필수.
