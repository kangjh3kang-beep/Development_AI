## 0. 설계 중심축 — "선택형 상세 분석을 전 시스템 기본으로"

사용자 지침을 플랫폼 전역 UX 철학으로 승격한다.

- **기본 동작 = 선택형**: 모든 분석 화면은 "분석 항목 선택(모듈 카탈로그)" 패널을 **기본 진입 상태**로 노출. 사용자가 필요한 모듈만 체크 → 선택분만 실행·과금.
- **자동 전체분석 = 옵션**: "한 번에 전체 분석(원클릭)"은 별도 버튼으로 제공하되 디폴트 아님.
- **재사용 공용 자산**: `AnalysisModuleSelector`(프론트 공용 컴포넌트) + `analysis_options` 표준 계약(백엔드)을 만들어 시장분석을 시작으로 부지·수지·인허가·ESG 등 **전 모듈이 동일 패턴 채택**.
- **투명성**: 선택수 → 예상 코인·예상 소요시간 실시간 표시. 미선택 모듈은 API 호출 자체를 생략(속도·비용 절감).

## 1. 비교분석 핵심 결론 (02·03 요약)

| 관점 | 시장 현황 | PropAI 기회 |
|---|---|---|
| 원클릭 사업성(주소→규모→분양가→ROI) | 국내 랜드북·랜드업·밸류맵 강세, 해외 TestFit/Deepblocks(공급측 중심) | 레드오션. 단독으론 차별화 약함 |
| **수요측 결합**(누가 살 것인가) | 사업성 툴 대부분 **공급측만**, 수요는 외부의존 | **최대 공백** — PropAI의 SGIS/KOSIS·실거래·상권·G2B 결합으로 초격차 |
| 인구이동→마케팅 페르소나 | 국내 자동화 사례 거의 없음(호갱노노 전입/전출 표시 수준) | **선점 가능**. 벤치마크 Esri Tapestry/Claritas PRIZM |
| 소득→적정 분양가 밴드 | 국내 미개척(KB PIR·HF HAI는 지표까지), 해외 HouseCanary가 affordability 예측 | **선점 가능**. PIR·DSR·LTV 역산 방법론·데이터 모두 공개 |
| 모듈러 크레딧 과금 UX | 국내 건당과금이 최선, 크레딧 UX 사례 없음. 해외 Deepblocks/HouseCanary 티어 | PropAI는 토큰계측·마진·코인·402 게이트 보유 → **즉시 차별화** |
| 근거·신선도·검증 노출 | 경쟁사는 수치만 노출 | public_data_registry·calculation_metadata·검증에이전트·해시체인으로 **PF/은행 제출 신뢰 우위** |

**전략 결론**: "공급측 원클릭"을 따라가되, **"수요측(인구이동·소득·가구) 결합 + 근거 가시화"** 를 핵심 차별화로 삼는다.

### 방법론 확정값(03)
- **인구이동 OD**: SGIS stats에 OD 엔드포인트 **없음** → 행안부(data.go.kr 15108093)/KOSIS(DT_1B26001_A01)에서 수집. 중력모형 `T_ij=α·(m_i^β·m_j^γ)/d_ij^δ`(δ 0.5~3).
- **분양가 밴드**: `타깃 가구 연소득 중위 × 목표 PIR` ↔ `DSR40%(스트레스금리)→PMT역산 대출→LTV→최대 분양가` 교차, 보수 하한 채택.
- **평형 MD**: 가구원수별 최저주거기준(하한) + 1인가구·연령분포 → 평형 추천(1인 33㎡↓/2~3인 59~75/4인 84/4인+ 99~115).
- **지역코드**: 법정동10/행정동10/SGIS 8/시군구5/집계구13. 행정동↔법정동 N:M → 이름매칭 금지·매핑테이블 join(PublicDataReader).
- **가명정보**: 결합전문기관 경유·집계 페르소나만, 개인단위 직접 마케팅 불가(개인정보보호법 28-2/28-3).

## 2. 사용자편의 극대화 상세 워크플로우 (To-Be)

### 2.1 메인 워크플로우 (선택형 기본)
```
[주소/필지 입력]
  → [① 분석 항목 선택 패널] ← 기본 진입 화면(선택형 디폴트)
       ☐ 실거래·시세 동향 (기본 필수, 무료/기본과금)
       ☐ 인구 이동·배후수요 (SGIS+행안부 OD)         [+N코인]
       ☐ 인구·가구 구조 (SGIS 센서스)                  [+N코인]
       ☐ 거시 소득·지불여력 (KOSIS)                    [+N코인]
       ☐ 적정 분양가 밴드 (PIR/DSR 역산) ★혁신          [+N코인]
       ☐ 사업 타당성 (규모·건축비·수익·ROI/NPV)         [+N코인]
       ☐ 마이크로 타겟팅 (K-Atlas)  🔒Locked(프리미엄)
       [예상 소모: 합계 N코인 · 예상 N초]   [⚡전체 자동분석]  [분석 시작]
  → [선택 모듈만 비동기 병렬 실행 + 진행 표시]
  → [결과 대시보드]
       · 핵심 재무지표 카드(예상 수익·ROI·NPV)
       · 인구 피라미드/가구 파이(Recharts) + 인구이동 히트맵(지도)
       · 적정 분양가 밴드(감당가능 X~Y억, 시행분양가 밴드 내/초과 판정)
       · AI 내러티브(타깃 페르소나·MD 제안) + 근거·출처·신선도·검증 배지
  → [보고서 내보내기(PDF/PPTX) — 선택 모듈 그대로 반영]
```

### 2.2 핵심 UX 원칙
- 선택 → 즉시 비용/시간 피드백(불확실성 제거).
- 모든 수치 옆 **근거 토글**(출처 API·기준시점·표본수·신뢰도). 데이터 없으면 "데이터 없음"(가짜값 금지).
- 잠금(Locked) 프리미엄은 더미수치 노출 금지 → "프리미엄 분석으로 해제" CTA만.
