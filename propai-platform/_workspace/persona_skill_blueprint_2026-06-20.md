# PropAI 실무 전문가 페르소나 분석 체계 — 설계 청사진 (2026-06-20)

> 목적: 개발사업 전과정을 **5대 실무 전문가 페르소나**(디벨로퍼·도시계획·설계·시공·분양대행)로
> 단계별 분석 → 실무 산출물 생성 → 결과물 검증까지 하나의 루프로 묶어, AI 에이전트가
> "분석 숫자"가 아니라 "실무 의사결정 + 실무 문서"를 만들도록 한다. **코드 무변경 설계문서**.

---

## 1. 목표와 차별점

- **As-Is**: 단계별 분석은 있으나(B5–B7 DAG), 결과가 분산된 "분석 값"이고 실무 문서·의사결정 게이트가 약함.
- **To-Be**: 각 단계를 **전문가 페르소나**가 주관해 ① 실무 의사결정 체크리스트로 판단 → ② 실무 문서(인허가 검토서·사업계획서·PF 자료·공사비 견적·분양계획서)로 산출 → ③ 결과물 검증 루프(작성→교차리뷰→trust 교차검증→정직고지) 통과.
- **차별점**: "분석"이 아니라 **"실무자가 실제로 만드는 결과물 + 그 검증"**.

## 2. 재사용 가능한 기존 자산 (백지 아님)

| 자산 | 페르소나 연결 |
|---|---|
| B5–B7 orchestration DAG(node-registry·dependency-graph·useOrchestrationStore·OrchestratorPanel) | 단계 골격 |
| 9종 interpreter(설계·수지·인허가·시장·부지 등) | 페르소나별 전담 해석 |
| `/expert-panel`(다관점 토론) + `/verify`(할루시네이션 검증) + trust.cross_validate | 결과물 검증 루프 |
| 심의엔진(deliberation, 타세션) | 도시계획/인허가 페르소나 audit |
| 다필지 통합분석(`/zoning/integrated-analysis`) | 토지·도시계획 단계 |
| 적정분양가·분양ERP·시장분석 | 분양대행 페르소나 |
| QTO·적산·공사비 | 시공 페르소나 |
| CAD/BIM·유닛믹스·법규검증 | 설계 페르소나 |
| 노드 불변계약(사실기반→interpreter→expert-panel→trust/verify→정직고지) | 검증 루프의 골격 |

→ 페르소나 = **흩어진 기능을 "전문가 역할 + 단계 + 검증"으로 오케스트레이션·문서화**하는 레이어.

## 3. 5대 전문가 페르소나 정의

각 페르소나 = **시스템 프롬프트(역할·말투·실무관점) + 실무 방법론 + 판단 기준(정량 KPI) + 산출물 템플릿 + 검증 항목**.

### 3-1. 디벨로퍼 (사업 총괄)
- **역할**: 사업 타당성·의사결정(Go/No-Go)·리스크·수익 총괄.
- **방법론**: 사업타당성(시장×입지×규모×수지) → 리스크 매트릭스 → 의사결정 게이트.
- **판단 KPI**: IRR·NPV·자기자본수익률(ROE)·DSCR·분양률 손익분기·LTV·사업비 회수기간.
- **산출물**: 사업계획서·투자심의서·리스크 리포트.
- **연결 기능**: 수지(/feasibility)·ROI·전문가패널·신뢰루프.

### 3-2. 도시계획 전문가 (인허가·개발방식)
- **역할**: 용도지역·지구단위·정비·개발방식 판정·인허가 전략.
- **방법론**: 용도지역/특이부지 분석 → 개발방식 판정(지구단위·도시개발·정비·가로주택·모아주택·역세권) → 인센티브(종상향·용적완화) → 인허가 로드맵.
- **판단 KPI**: 건폐/용적 한도(법정/조례/실효)·종상향 가능성·기부채납률·인허가 소요기간·리스크 등급.
- **산출물**: 인허가 검토서·개발방식 비교표·인허가 로드맵.
- **연결 기능**: 인허가AI·개발방식시뮬·규제대시보드·다필지통합분석·심의엔진·특이부지게이트.

### 3-3. 설계사 (건축 계획)
- **역할**: 배치·평면·법규·디자인·매스.
- **방법론**: 배치계획 → 유닛믹스 최적화 → 법규검토(건축법·주차장법·일조·피난) → 매스스터디(CAD/BIM).
- **판단 KPI**: 건폐/용적 실현율·세대수·전용률·주차대수 충족·일조 적합·평형 구성.
- **산출물**: 기본설계도·법규 체크리스트·유닛믹스표·CAD/BIM 모델.
- **연결 기능**: CAD/BIM 스튜디오·유닛믹스 최적화·법규검증(building-compliance)·설계 interpreter.

### 3-4. 시공사 실무자 (시공·원가)
- **역할**: 공법·공정·원가·QTO·안전.
- **방법론**: 공법 선정 → 공정 계획 → QTO/적산(부위별 물량) → 원가관리 → 안전계획.
- **판단 KPI**: 공사비 평단가·총공사비·공기·원가율·간접비율·안전등급.
- **산출물**: 공사비 견적서·QTO 물량내역·공정표·안전계획서.
- **연결 기능**: QTO·적산(5D)·공사비(/cost/estimate-overview)·BIM 적산.

### 3-5. 분양대행 전문가 (분양·마케팅)
- **역할**: 분양가·전략·청약·계약·마케팅.
- **방법론**: 적정분양가(거래사례비교 + 지불여력 PIR/DSR/LTV 검증) → 분양전략 → 청약 분석 → 계약관리 → 마케팅.
- **판단 KPI**: 적정분양가(평단가)·예상 분양률·청약경쟁률·계약률·중도금 구조·분양수입.
- **산출물**: 분양계획서·분양가 책정서·청약 전략·계약 시뮬레이션.
- **연결 기능**: 적정분양가·분양ERP(가격엔진)·시장분석·실거래지도·분양정보 모니터링.

## 5. 결과물 검증 루프 (사장님 강조 "검증까지의 전체 루프")

각 단계 산출물은 다음 4단 게이트를 통과해야 "확정"으로 표기, 미통과는 "잠정/미확보 정직고지":

1. **작성**: 전담 페르소나가 사실 데이터(공공데이터 실값) 기반으로 산출물 작성.
2. **교차 전문가 리뷰**: 인접 페르소나(예: 수지↔분양가↔공사비)가 상호 검토(/expert-panel 재사용).
3. **trust 교차검증**: trust.cross_validate로 원천 데이터 이상치·일관성 검증(예: 분양가 vs 시장, 공사비 vs 표준).
4. **정직고지**: 미확보·잠정·가정은 명시(무목업). 특이부지·게이트 미통과는 확정% 억제.

→ 이미 노드 불변계약(사실기반→interpreter→expert-panel→trust/verify→정직고지)이 이 루프의 골격. 페르소나는 여기에 **역할·체크리스트·산출물 템플릿**을 부여.

## 6. 기술 구현 방안

- **백엔드**: `persona_registry`(5 페르소나 = 시스템프롬프트 + 실무 체크리스트 + 산출물 JSON 스키마). 기존 interpreter를 페르소나로 래핑하거나 전담 interpreter 부여. `/persona/{key}/analyze`(단계·컨텍스트 입력 → 산출물 + 검증 결과). 기존 expert-panel/verify/trust 연결.
- **단계 매핑**: 기존 node-registry에 `persona` 필드 추가(또는 신규 `stage-persona-map`). 가산·무변경.
- **산출물**: 구조화 출력(실무 문서 템플릿) → 프론트 카드 + PDF(기존 reportlab/docx).
- **프론트**: 단계별 "페르소나 결과물 카드" + "검증 배지"(pass/잠정/미확보). OrchestratorPanel 확장.
- **과금**: LLM 산출물은 use_llm 옵션·관리자 설정(미설정 무료). 규칙기반 체크리스트는 무과금.

## 7. 단계적 구현 로드맵

- **P0 (이 문서)**: 설계 청사진 — 사장님 검토·합의.
- **P1 파일럿**: persona_registry 골격 + **분양대행 페르소나 1개**를 끝까지(분양가 책정→전략→청약→계약 체크리스트 + 산출물 + 검증). 가치 빠른 실증.
- **P2**: 나머지 4개 페르소나(디벨로퍼·도시계획·설계·시공).
- **P3**: 결과물 검증 루프 통합(교차리뷰·trust·정직고지 전 페르소나).
- **P4**: 산출물 실무 문서화(사업계획서·인허가검토서·PF자료·분양계획서 PDF) + 단계 오케스트레이션(OrchestratorPanel 페르소나 모드).

## 8. 절대 경계 (전 구현 공통)

- useProjectContextStore **additive·읽기소비**, persist version/migrate·purifyPersistedContextState·addressTokenMismatch·snapshots(계정격리) **미접촉**.
- auto_zoning_service(land runner)·deliberation 타세션 소유 **미접촉(읽기참조만)**.
- **무목업**(미확보 정직표기)·**무과금**(관리자설정·미설정 무료, 신규 항목 기본0/module_fees).
- 멀티세션: 공유파일(main.py·node-registry) 편집 전 coord claim, 전용 워크트리.
- 성장루프: 각 페르소나 구현 → fresh 코드리뷰 ≥4.5 별도패스 → 배포 → 라이브검증.

## 9. 열린 결정 사항 (사장님 합의 필요)

1. **파일럿 페르소나**: 분양대행(사장님 강조) vs 도시계획(인허가가 사업 좌우) — P1 시작점.
2. **산출물 깊이**: 화면 카드까지 vs 실무 PDF 문서까지(P1 범위).
3. **단계 오케스트레이션 UI**: 기존 OrchestratorPanel 확장 vs 신규 "전문가 워크플로우" 화면.
4. **페르소나 interpreter**: 기존 9 interpreter 래핑 vs 페르소나 전담 신규.
