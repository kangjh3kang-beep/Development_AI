## 핵심: 지도 클릭 = 단일 팝오버(컨텍스트 메뉴) — 컴프 "단일 팝오버" 구현

**동작 계약**
1. 지도 클릭(마커 아님) → 그 좌표에 팝오버 1개(기존 팝오버는 자동 대체 — 동시 2개 금지).
2. 팝오버 구성(위→아래):
   - 헤더: 클릭 지점 요약 — 지번 주소(리버스 지오코딩 기존 경로)·면적·용도지역(로드되면)
   - 액션 리스트:
     a. **➕ 이 필지 선택** / ➖ 선택 해제(이미 선택 시) — 기존 필지선택 API 재사용
     b. **ℹ️ 필지 정보 보기** — 주소·PNU·면적·용도지역·지목 상세(기존 메타)
     c. **📏 거리재기 시작** — 측정 모드 토글(클릭 연속점 폴리라인+거리 라벨, ESC/더블클릭 종료)
     d. 근처 색깔점 있으면: **● 주변 데이터 N건** — 해당 마커 정보 리스트
   - 푸터: **데이터 출처**(VWorld·MOLIT 등) mono 캡션 — 컴프 "팝업 출처 푸터"
3. 닫힘: 다른 곳 클릭·ESC·지도 드래그 시작·액션 실행 후(선택 액션은 토스트/카드 반영으로 피드백).
4. 측정 모드 중에는 팝오버 억제(클릭=측정점 추가).

**즉시선택→팝오버 전환의 회귀 방지**: 기존 "지도 클릭=바로 필지선택" 사용자 습관 고려,
팝오버 최상단 액션(선택)을 클릭 1회 추가로 수행 가능하게 — 총 2클릭. 대신 오선택(U6)·
정보 접근·도구 진입이 전부 한 제스처로 통합됨(순편익 양).
