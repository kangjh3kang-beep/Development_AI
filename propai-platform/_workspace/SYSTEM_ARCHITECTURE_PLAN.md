### 2-2. 모듈 간 인터페이스 정의 (핵심)

```python
# ── STEP 1 → STEP 2 전달 데이터 ──
class SiteToDesignPayload:
    """부지분석 결과 → 설계 모듈 입력"""
    pnu_codes: list[str]                 # 필지 PNU
    zone_type: str                       # 용도지역 (제1종일반주거 등)
    max_bcr: float                       # 최대 건폐율 %
    max_far: float                       # 최대 용적률 %
    max_height: float                    # 최대 높이 m
    setback_front: float                 # 전면 이격거리 m
    setback_side: float                  # 측면 이격거리 m
    land_area_sqm: float                 # 대지면적
    land_shape: GeoJSON                  # 필지 형상 (GeoJSON Polygon)
    road_width: float                    # 접도 너비 m
    official_land_price: float           # 공시지가 원/㎡
    nearby_transactions: list[dict]      # 인근 실거래가 목록
    sunlight_constraints: dict           # 일조권 사선 제한

# ── STEP 2 → STEP 3 전달 데이터 ──
class DesignToCostPayload:
    """설계 결과 → 공사비 모듈 입력"""
    total_gfa_sqm: float                 # 연면적
    floor_count_above: int               # 지상 층수
    floor_count_below: int               # 지하 층수
    structure_type: str                  # 구조 (RC/SRC/S)
    building_type: str                   # 건물유형 (아파트/오피스텔/상가 등)
    unit_mix: list[UnitMixItem]          # 평형별 세대수
    ifc_model_url: str | None            # IFC 파일 URL (있으면)
    material_quantities: list[MaterialQty]  # 자재 물량 (IFC에서 추출)
    svg_drawings: list[str]              # 설계 도면 ID 목록

# ── STEP 3 → STEP 4 전달 데이터 ──
class CostToFeasibilityPayload:
    """공사비 결과 → 수지분석 입력"""
    total_construction_cost: float       # 총 공사비
    cost_by_category: dict[str, float]   # 공종별 비용
    material_quantities: list[dict]      # 자재 물량 (ESG용)
    construction_months: int             # 공사기간
    unit_construction_cost_per_pyeong: float  # 평당 공사비
```

### 2-3. 핵심 기술 스택

| 계층 | 기술 | 용도 | 참조 |
|------|------|------|------|
| **BIM 파싱** | IfcOpenShell | IFC→물량 자동산출 | ISO 16739, buildingSMART QTO |
| **GIS/공간** | GeoPandas + V-World API | 필지 형상/용도지역 | 국가공간정보포털 |
| **설계 최적화** | SciPy SLSQP + NSGA-II | 유닛믹스/배치 최적화 | ArkDesign 벤치마크 |
| **법규 엔진** | RASE 방법론 + Custom DSL | 건축법→논리식 변환 | Solibri, KBim 참조 |
| **LCA 엔진** | ISO 14040 + ICE DB | 자재→탄소 매핑 | One Click LCA 참조 |
| **리스크** | NumPy Monte Carlo | 수익률 확률분포 | ARGUS Enterprise 참조 |
| **오케스트레이터** | LangGraph (기존) | 파이프라인 상태머신 | 이미 구현됨 |
| **보고서** | ReportLab + Jinja2 | 은행 PF 보고서 | Feasibly 참조 |

## 3. 단계별 구현 로드맵

### Phase 1: 데이터 파이프라인 구축 (최우선, 2주)

**목표**: "주소 입력 → 7단계 자동 분석" 원클릭 파이프라인

| STEP | 작업 | 핵심 변경 | 난이도 |
|------|------|----------|--------|
| 1-1 | **ProjectPipeline 서비스 신규 작성** | `apps/api/app/services/pipeline/project_pipeline.py` — 7단계 순차 실행 + 상태 관리 | 높음 |
| 1-2 | **부지분석→설계 인터페이스** | SiteToDesignPayload 정의, 부지분석 결과를 설계 모듈 입력으로 자동 매핑 | 중간 |
| 1-3 | **설계→공사비 인터페이스** | DesignToCostPayload 정의, 설계 결과(면적/층수/구조)를 공사비 입력으로 자동 매핑 | 중간 |
| 1-4 | **공사비→수지분석 인터페이스** | CostToFeasibilityPayload 정의, 공사비 결과를 수지분석 자동 입력 | 낮음 (일부 존재) |
| 1-5 | **공사비→ESG 인터페이스** | 자재 물량 목록을 LCA 엔진에 자동 전달 | 중간 |
| 1-6 | **파이프라인 API 엔드포인트** | `POST /api/v1/projects/{id}/run-pipeline` — 원클릭 트리거 | 낮음 |
| 1-7 | **프론트엔드 파이프라인 UI** | 진행 상태 표시 + 단계별 결과 확인 | 중간 |

### Phase 2: 모듈별 고도화 (4주)

#### 2-A. 부지분석 고도화 (L1→L3)

| 작업 | 내용 | 참조 |
|------|------|------|
| 필지 경계 자동 수집 | V-World 연속지적도 API → GeoJSON 폴리곤 | 스페이스워크 랜드북 |
| 인접 필지 분석 | 도로 접면, 인접 건물 높이 | 세움터 건축물대장 |
| 실거래가 자동 수집 | MOLIT API 실거래가 → 시세 추정 | 랜드업 |
| 일조권 사선 계산 | DEM 기반 + 인접 건물 → 일조 시뮬레이션 | ArkDesign |
| 결과를 설계 모듈에 자동 전달 | SiteToDesignPayload 생성 → Pipeline | 자체 설계 |

#### 2-B. 건축설계 고도화 (L2→L3)

| 작업 | 내용 | 참조 |
|------|------|------|
| 법규 실시간 검증 | 건폐율/용적률/높이/일조/주차 자동 체크 | Solibri, KBim |
| 유닛믹스 최적화 | SLSQP 기반 평형 배분 → 수익 극대화 | ArkDesign (이미 부분 구현) |
| 설계→IFC 변환 | SVG 도면 → 간이 IFC 모델 자동 생성 | IfcOpenShell |
| 설계 결과→공사비 자동 전달 | DesignToCostPayload 생성 → Pipeline | 자체 설계 |

#### 2-C. 공사비 고도화 (L3→L3+)

| 작업 | 내용 | 참조 |
|------|------|------|
| 설계 데이터 자동 수신 | 설계 모듈에서 면적/층수/구조 자동 수신 | Pipeline 연동 |
| 도면 없이도 개산견적 | 건물유형+연면적+층수 → 유사 사례 기반 견적 | KICT 연구 |
| IFC 업로드 UI | 프론트엔드 도면 업로드 → 물량 자동산출 | CostX 참조 |
| 시장단가 DB 연동 | 한국물가정보/조달청 단가 자동 업데이트 | Cubicost 참조 |
| 공사비→ESG 자재 전달 | 자재 물량 목록 → LCA 엔진 자동 | One Click LCA |

#### 2-D. ESG/탄소 고도화 (L2→L3)

| 작업 | 내용 | 참조 |
|------|------|------|
| 자재-탄소 DB 구축 | ICE Database + 환경부 EPD DB 연동 | One Click LCA |
| 공사비 자재 자동 매핑 | 공종코드 → 자재 → 탄소계수 자동 체인 | EN 15978 |
| GRESB 자동 스코어링 | 30% Management + 70% Performance 자동 산출 | GRESB 2025 기준 |
| G-SEED 인증 시뮬레이션 | 한국 녹색건축 인증 점수 자동 예측 | KICT |
| 개선 시나리오 제안 | 자재 대체 → 탄소 저감 효과 자동 계산 | EC3 |

#### 2-E. 투자수익성 고도화 (L3→L3+)

| 작업 | 내용 | 참조 |
|------|------|------|
| 자동 현금흐름 생성 | 수지분석 결과 → 월별 현금흐름 자동 생성 | ARGUS Enterprise |
| 시장 데이터 자동 연동 | 실거래가+분양가+금리 → 변수 자동 설정 | CoStar, R-ONE |
| Tornado 민감도 분석 | 각 변수 ±10% 변동 → NPV/IRR 영향 순위 | Feasibly |
| LP/GP 분배 모델 | Waterfall 수익 배분 구조 | ARGUS |

#### 2-F. 인허가 고도화 (L0→L2)

| 작업 | 내용 | 참조 |
|------|------|------|
| 법규 룰엔진 구축 | 건축법 핵심 조항 → RASE 방법론으로 논리식 변환 | Hjelseth & Nisbet (2011) |
| 설계안 자동 검증 | 설계 결과 → 법규 룰 순차 검증 → PASS/FAIL | Solibri, SMC |
| 인허가 서류 자동 생성 | 설계설명서, 에너지절약계획서 템플릿 자동 작성 | 세움터 |
| 법규 변경 모니터링 | 법제처 API → 프로젝트 영향 자동 알림 | 기존 코드 강화 |

### 현재 PropAI의 강점 (유지·강화)
1. **수지분석 엔진** (Level 3) — 14개 엔드포인트, 4개 개발유형, 버전관리

4. **은행 PF 보고서** — 경쟁사 없는 차별화 기능

### 최대 Gap (최우선 해결)
1. **모듈 간 자동 데이터 파이프라인** — 현재 섬 구조 → 혈관 네트워크로 전환

3. **공사비→ESG 연결** — 자재 물량 자동 전달 미구현
4. **인허가 법규 엔진** — Level 0에서 L2로 끌어올려야 함
