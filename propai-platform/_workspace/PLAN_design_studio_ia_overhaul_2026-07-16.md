# 설계 스튜디오 IA 전면 재설계 계획 (2026-07-16)

사용자 지시: "불필요한 중복정보·필드·패널 제거 + 화면구성/인터페이스 전반 개선 + 갇힌 프레임 해결 + 디자인 완전 리팩토링(디자인지침 부합) + 가독성·직관·효율 + 정보출력·기능 100% + 생각지 못한 개선 반영".

## 0. 실행 순서 (충돌 방지 — 필수)
- **Phase 1 (진행 중)**: `fix/design-studio-readability-ssot` — 확정 5결함(AI의견 마크다운·KPI/한도 SSOT·헤더 용도지역 전파·준비상태 술어 통일·과적 완화). 같은 파일(DesignStudio/DesignWorkspace/MetricBar)을 만지므로 **먼저 머지**.
- **Phase 2 (본 계획)**: IA 전면 재설계 — Phase 1 머지 후 최신 main 기준 전용 워크트리에서 구현. 성장루프(구현→적대리뷰→R2→CI→머지→배포).

## 1. 진단 — 중복·낭비 전수 목록 (라이브 스크린샷 근거)

### D1. 같은 정보의 다중 표기
| 정보 | 표기 위치 | 중복도 |
|---|---|---|
| 분석 주소 | ①상단 헤더바 "주소" ②우측 레일 "분석 주소" ③콘솔 "부지분석 연동: …" | **3중** |
| 대지면적 | ①헤더바 ②확정 칩 ③"직접 조정(고급)" 서랍 입력필드 | **3중(동시 노출)** |
| 용도지역·건물용도 | ①확정 칩 ②서랍 입력필드 (+헤더바는 "—"로 끊김 — Phase 1 수정) | 2중(동시) |
| 파이프라인 단계 | ①좌측 레일(1차/2차/3차 카드) ②하단 스테퍼(①—②—③) ③우측 레일 안내문("부지 기준 확정 → Top-N 개요 → 도면 편집 순서") | **3중** |

### D2. 공간 낭비
- **우측 레일(~25% 폭)**: "분석 주소 1줄 + 안내문 2줄"만 표시. 나머지 전부 여백.
- **잠긴 단계(2차·3차) 화면**: 자물쇠 아이콘+문구 1개가 전체 뷰포트 소모. 안내가치 대비 공간 극낭비.
- **좌측 레일 하부**: 단계 카드 3개 이후 전부 여백("엔진 파이프라인 L1-L5" 접힘만 하단에).

### D3. 오해 유발
- 하단 KPI바가 **미생성 상태**인데 "설계 산출 · 생성 결과 KPI"로 법정 기본값(20%/100%/12,079/4층) 표시 — 생성 결과가 아님(Phase 1에서 라벨 정직화, Phase 2에서 상태 분기).
- "직접 조정(고급)" 서랍이 열리면 칩과 서랍이 같은 값을 동시에 두 벌 노출 — 어느 쪽이 정본인지 혼란.

### D4. 갇힌 프레임 (구조)
- 중앙 뷰포트 고정높이(`DesignWorkspace.tsx:391` `lg:h-[calc(100dvh-15rem)]`) + 내부 스크롤 → 페이지 스크롤과 이중 스크롤.
- DesignStudio 2열 분리 임계(@4xl=896px)가 3열 셸 중앙폭(~760px)에서 미발화 → 10+섹션 세로 스택.

## 2. 개선안 (IA 재설계)

### A. 중복 제거 — 표면별 단일 책임
- **파이프라인 표현 = 좌측 레일 하나로 일원화.** 하단 스테퍼 제거(레일 접힘 시에만 나타나는 컴팩트 인디케이터로 대체). 우측 레일의 순서 안내문 제거.
- **식별 정보 = 상단 ContextHeader 정본.** 우측 레일의 "분석 주소" 카드 제거(헤더와 완전 중복). 콘솔 연동문구는 상태 배지 1줄로 축소.
- **칩↔서랍 동시 노출 제거**: 서랍 열림 시 칩 숨김(또는 칩=서랍 토글 버튼화). 같은 값 2벌 동시 노출 금지.

### B. 우측 레일 재정의 → "근거·한도 패널"
빈 공간을 가치 정보로 채움(전부 기존 데이터 재배치 — 신규 API 불필요):
1. **실효 한도 카드**: 적용 건폐율/용적률/층수(클램프)·산출 근거(법령 링크 — 기존 "근거 보기" 내용 인라인화).
2. **경고·특이사항**: 특이부지·신뢰도 낮음 경고를 여기로 상시 고정(콘솔 스크롤에 묻히지 않게).
3. **다음 단계 CTA**: 현재 단계 완료 조건 + 다음 행동 버튼.
- 대안: 우측 레일 기본 접힘 → 중앙 폭 확보로 2열(@4xl) 자연 발화. 두 안 중 구현 시 실측으로 결정.

### C. 잠긴 단계 화면 재설계 (빈 화면 → 준비 대시보드)
자물쇠 문구 대신:
1. **해제 요건 체크리스트**(실데이터): 주소 ✓ / 용도지역 ✓ / 대지면적 ✓ — 충족 항목 실시간 체크, 미충족 항목에 바로가기 버튼.
2. **이 단계의 산출물 미리보기**(정적 스켈레톤): Top-N 카드 구조/도면 편집실 구조를 회색 스켈레톤으로 — 무엇을 얻게 되는지 예고(날조 아님·"예시 구조" 라벨).
3. 요건 전부 충족 시 자동 언락 + 시각 전환.

### D. 프레임 해방 — 단일 스크롤 전환
- 중앙 고정높이+내부 스크롤 제거 → **페이지 단일 스크롤**. 좌측 레일만 sticky(top 고정), 하단 KPI/FlowAdvanceBar는 sticky bottom 유지. 이중 스크롤 소멸 = "갇힌 프레임" 근본 해소.
- v420 흐름 IA(FlowAdvanceBar·stateWord·flowTerminal=view==="draw"만) 회귀 금지.
- 컨테이너 쿼리 유지. 우측 레일 접힘/재정의와 결합해 중앙 실폭 900px+ 확보 시 인스펙터/캔버스 2열 자연 발화.

### E. KPI바 상태 분기 (정직화 완성)
- 미생성: "설계 산출 대기 — 추천안을 생성하면 결과 KPI가 표시됩니다" + 법정 기준값은 "법정상한" 배지로 구분(Phase 1 연계).
- 생성 후: 생성 결과 KPI 정상 표시(현행).

### F. 생각지 못한 개선(추가 반영)
1. **레일 자동 접힘**: 뷰포트 폭이 좁으면 우측 레일 자동 접힘 → 중앙 우선(사용자 수동 복원 가능).
2. **전문 필드 쉬운 설명**: 층고(M)·용적률 등에 ⓘ 툴팁(기존 "쉬운 설명" 톤 재사용).
3. **단계 완료 시각 피드백**: 조건 충족 순간 레일 카드 체크 애니메이션 + 다음 단계 하이라이트(물 흐르듯 진행 강화).
4. **접근성**: 잠금 상태 aria-disabled·스크린리더 문구.
