# PropAI 하네스 엔지니어링 v2 — 근본 재설계 계획

## 핵심 원칙

> "주소(다필지) 입력 → 토지정보 수집 → 법규/조례 분석 → 시장 조사 → 수지분석"
> 이 파이프라인이 모든 모듈의 기초 데이터가 되며, 일관성과 검증 가능성을 보장한다.

---

## LAYER 0: 기초 데이터 파이프라인 (모든 모듈의 근간)

### 0-1. 다필지 주소 입력 시스템

**현재 문제**: 단일 주소만 입력 가능, 합필/다필지 분석 불가

**구현**:
- 주소 입력 시 여러 필지(PNU) 동시 입력 지원
- 각 PNU별 토지정보 병렬 수집
- 합필 시 면적 합산, 접도 조건 통합 분석
- 결과를 `ProjectLandData` 스토어에 저장 → 모든 모듈에서 참조

```
입력: ["의정부시 의정부동 224", "의정부시 의정부동 225"]
  ↓ (병렬 PNU 조회)
출력: ProjectLandData {
  parcels: [
    { pnu: "4115010100102240000", area: 500, jimok: "대", ... },
    { pnu: "4115010100102250000", area: 300, jimok: "대", ... },
  ],
  total_area_sqm: 800,
  merged_zone_type: "제2종일반주거지역",
  ...
}
```

### 0-2. 토지정보 수집 파이프라인 (1회 수집 → 전 모듈 공유)

**현재 문제**: 각 모듈이 개별적으로 API 호출, 데이터 불일치

**구현**: `/zoning/comprehensive` 1회 호출로 아래 전부 수집

| 수집 항목 | API 소스 | 저장 위치 |
|----------|---------|----------|
| PNU, 좌표 | VWORLD 지오코딩 | parcels[] |
| 지목, 면적, 소유구분 | VWORLD 필지 | parcels[].land_register |
| 개별공시지가 (연도별) | VWORLD NED | parcels[].official_price |
| 용도지역/지구/구역 | VWORLD NED 토지이용계획 | land_use_plan |
| 행위제한 상세 | 토지이용계획 + 규제 매핑 | regulations[] |
| 건축물대장 (기존건물) | 공공데이터포털 | existing_buildings[] |
| 지자체 조례 건폐율/용적률 | 법제처 API | ordinance |
| 법정 상한 건폐율/용적률 | 내부 ZONE_LIMITS | national_limits |

**핵심**: 이 데이터가 수집되면 `ProjectLandData`에 저장되고,
이후 수지분석/설계/법규검토/BIM 모든 모듈이 이 데이터를 참조.

### 0-3. 데이터 검증 레이어 (할루시네이션 방지)

**현재 문제**: 데이터 출처 불명, 검증 없음

**구현**:
- 모든 수치에 출처 태그 부착: `{ value: 60, source: "법제처API", fetched_at: "2026-05-29" }`
- 법정 상한 ↔ 조례값 교차 검증 (조례 > 법정이면 경고)
- 공시지가 ↔ 실거래가 괴리율 자동 검증
- 검증 실패 시 사용자에게 경고 표시

## LAYER 1: 매출 산정 엔진 (시장조사 기반)

### 1-1. 주변 시세 조사

**현재 문제**: 고정 분양가 테이블 (경기=2500만원/평 하드코딩)

**구현**:
```
주소 입력
  ↓
[1] 인근 실거래가 수집 (반경 1km, 최근 1년)
    - 아파트: 국토부 실거래가 API
    - 오피스텔/상가: 추후 API 추가 시
    ↓
[2] 유형별 시세 분석
    - 평균 시세, 최고/최저, 추세 (상승/하락)
    - 신축 프리미엄 계수 (신축 vs 기존 대비)
    ↓
[3] 공급가 3단계 산정
    - 보수적: 주변 시세 × 0.85 (미분양 리스크 반영)
    - 안정적: 주변 시세 × 0.95
    - 공격적: 주변 시세 × 1.10 (브랜드 프리미엄)
    ↓
[4] 개발유형별 매칭
    - 공동주택(일반분양): 아파트 시세 기준
    - 지역주택조합: 조합원 분담금 방식
    - 재개발/재건축: 비례율 + 추가부담금 방식
    - 오피스텔: 소형 오피스텔 시세 기준
    - 도시형생활주택: 원룸/투룸 시세
```

### 1-2. 공급물량 산정 (법규 기반)

**현재 문제**: 법정 용적률만 적용, 조례 미반영

**구현**:
```
토지면적 × 유효 용적률(조례 기준) = 최대 연면적
  ↓
용도별 면적 배분 (주거/상가/주차 등)
  ↓
세대수 산정 = 주거면적 ÷ 평균 세대면적
  ↓
분양면적 = 전용면적 + 공용면적 (서비스율 적용)
```

유효 용적률 결정 순서:
1. 지자체 조례값 (법제처 API 실시간)
2. 법정 상한 (국토계획법)
3. 개발유형별 통상 용적률 (min)

## LAYER 2: 지출 산정 엔진 (보수적 원칙)

### 2-1. 토지비

| 항목 | 산정 방법 |
|------|---------|
| 토지매입비 | 공시지가 × 시가반영율 또는 감정평가액 |
| 취득세 | 매입가 × 4.6% (법인) 또는 1~3% (개인) |
| 등록세 | 매입가 × 0.2% |
| 법무사 비용 | 건당 정액 |
| 중개수수료 | 매입가 × 0.4~0.9% |
| 명도비/이주비 | 기존 건물 세입자 수 × 단가 |

### 2-2. 공사비

| 항목 | 산정 방법 |
|------|---------|
| 건축공사비 | 연면적 × 구조별 단가 (RC: 240만/m², SRC: 280만 등) |
| 철거비 | 기존 건물 연면적 × 15만/m² (건축물대장 연동) |
| 토목공사비 | 대지면적 × 20만/m² |
| 조경공사비 | 대지면적 × 5% × 공사비 |
| 부대공사비 | 건축공사비 × 3~5% |
| 설계비 | 건축공사비 × 4~6% |
| 감리비 | 건축공사비 × 2~3% |

### 2-3. 금융비

| 항목 | 산정 방법 |
|------|---------|
| 브릿지론 이자 | 토지비 × 대출비율 × 연이율 × 기간/12 |
| PF 이자 | 총사업비 × 대출비율 × 연이율 × 기간/12 |
| 중도금 이자 | 분양대금 × 중도금율 × 연이율 × 기간/12 |
| 보증수수료 | 분양보증: HUG 보증료, PF보증: 신보/기보 |

### 2-4. 기타 비용

| 항목 | 산정 방법 |
|------|---------|
| 분양 관련 | 모델하우스, 광고비, 분양대행 수수료 |
| 부담금 | 학교용지부담금, 기반시설부담금, 광역교통시설부담금 |
| 인허가 | 교통영향평가, 환경영향평가, 건축심의 등 |
| 제세공과 | 재산세, 종합부동산세, 법인세 |
| 예비비 | 총사업비 × 3~5% (보수적) |

## LAYER 3: 수지분석 종합

### 3-1. 시나리오 분석

```
[보수적 시나리오]
  매출: 주변시세 × 0.85
  지출: 모든 항목 + 예비비 5%
  분양율: 80%

[안정적 시나리오]
  매출: 주변시세 × 0.95
  지출: 모든 항목 + 예비비 3%
  분양율: 90%

[공격적 시나리오]
  매출: 주변시세 × 1.10
  지출: 최적화 항목 + 예비비 2%
  분양율: 100%
```

### 3-2. 수익성 지표

- 사업수익률 = (총수입 - 총지출) / 총지출 × 100
- 자기자본수익률(ROE) = 순이익 / 자기자본 × 100
- IRR (내부수익률): 현금흐름 기반
- NPV (순현재가치): 할인율 8% 기준
- BEP (손익분기 분양율): 총지출 / 총수입 × 100

## 구현 순서

### STEP 1 (이번 세션): 기초 데이터 파이프라인 완성
- [ ] ProjectLandData 스키마 정의
- [ ] /zoning/comprehensive 응답에 전체 데이터 포함 확인
- [ ] 프론트엔드: 입지분석 결과를 ProjectContextStore에 완전 저장
- [ ] 모든 모듈이 ProjectLandData를 참조하도록 데이터 흐름 통일

### STEP 2: 매출 엔진 재설계
- [ ] 주변 실거래가 수집 + 통계 분석 (평균/최고/최저/추세)
- [ ] 보수적/안정적/공격적 3단계 공급가 자동 산정
- [ ] 개발유형별 매출 계산 로직 교차 검증

### STEP 3: 지출 엔진 재설계
- [ ] 토지비: 공시지가 연동, 취득세 자동 계산
- [ ] 공사비: 구조별/용도별 단가 DB, 건축물대장 철거비 연동
- [ ] 금융비: 브릿지/PF/중도금 이자 자동 계산
- [ ] 기타비: 부담금/인허가/제세공과 체크리스트

### STEP 4: 검증 레이어
- [ ] 모든 수치에 출처 태그 부착
- [ ] 법정상한 ↔ 조례 교차 검증
- [ ] 공시지가 ↔ 실거래가 괴리율 검증
- [ ] 수익률 합리성 검증 (±100% 초과 시 경고)
