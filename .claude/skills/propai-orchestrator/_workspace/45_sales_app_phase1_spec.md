# 분양 현장앱 — Phase 1 통합 구현명세 (대표 공동설계, 2026-06-06)

아키텍처 결정: **하이브리드(RLS shared-schema + 대형만 분리) · Redis Streams · 국소 분산락 · WebSocket/SSE · 해시체인 무결성** (리서치 검증, _workspace/44).
전제: **Phase 0(RLS 보안토대) 선결** 후 Phase 1. 현 v62 분양ERP 88테이블·11모듈·조직도·수수료·무결성가드·통합모니터링 이미 보유(~80%).

## A. 인증·접근 워크플로우
- 앱 실행 → ① 개인 SSO(전역 계정) → ② **본인 멤버십 현장 리스트** → ③ **현장별 2차 비밀번호(site_passwords)** → 현장 세션토큰 → ④ 현장 역할(시행/대행/본부장/팀장/직원) 기반 메뉴·기능·데이터 차등.
- 3중 방어: RLS(데이터 p_site/p_org) + require_role(기능) + 프론트 역할 게이팅(UI, 메인 페르소나게이팅 패턴 재사용).
- 신규: site_passwords 테이블·2차인증 플로우, GET /my-sites(멤버십 현장), 역할별 메뉴맵.

## B. 서브도메인 PWA (설치형 현장앱)
- `*.4t8t.net` 와일드카드 DNS + 테넌트 미들웨어(deps_sales 호스트파싱 존재→배선). 현장 생성→siteCode.4t8t.net.
- 현장별 **PWA manifest 동적**(사통팔땅 아이콘 통일 + 현장 name/start_url/scope). 오리진별 독립 설치.
- 현장 생성→"앱 열기(새창)/설치 안내" CTA. 기존 SW(v55)·manifest·아이콘 토대 재사용.

## C. 공유·배포 (바이럴)
- 공유 링크 + **QR**(방문데스크 mh_desks QR 결합) + Web Share API + 카카오알림톡(notify.py 보유).
- **MGM 추천코드**: 공유링크에 상담사 추천코드 → 설치·계약 추적 → 조직도·수수료 자동귀속.
- iOS 설치 가이드(사파리 수동단계 툴팁). 정보통신망법(수신동의·야간제한·거부) 준수.

## D. 고객관리 (현장별 + 통합) + 업무일지 + 문자
- 토글: [현장별](그 현장+내 역할범위) / [통합](내 멤버십 전 현장 union, 각 현장 역할범위 적용). 현장칩 필터·온도·계약단계 정렬.
- 통합 요약=개인범위 허용, 민감상세=현장 2차인증 후. 신규 GET /sales/my-customers?scope=all|site.
- ★**고객관리카드 히스토리**(상담·방문·계약단계 타임라인) + **카드 내 문자/알림톡 발송**(notify.py 보유, 발송이력 카드기록). **업무일지**(일일활동→고객이력·실적 연계). 문자=정보통신망법(수신동의·발신번호 사전등록).

## E. 직원관리 + 구인구직 (프리랜서) + 공통 마켓
- 대행/본부장/팀장: **현장별 직원관리 + 종합 직원관리**(여러 현장 뛰는 프리랜서 통합). 실적·출근·계약·수수료.
- ★**공통(public) 마켓플레이스**(현장 격리 아님): 구인/구직 · 분양현장 홍보 · 대행사 모집을 플랫폼 공통 컨텐츠로. 검색·필터(지역/전문분야/실적)·매칭.
- ★**재사용 프로필**(잡코리아/링크드인 패턴): **개인 프로필**(상담사 경력·지역·전문분야·실적·자격·희망조건) + **회사 프로필**(대행사/시행사 소개·규모·진행현장·평판) 1회 작성·저장 → 구인공고 등록·구직 신청·현장 홍보 시 **불러오기 원클릭**.
- 연계: 프로필→채용시 조직도 현장 멤버십 자동연결 + 추천코드 + 소셜그래프. 분양현장 홍보=B2C(고객유치)+B2B(대행유치) 양면.
- 정직: 프로필 공개범위/연락처 마스킹, 허위검증, 위촉계약 법적고려, 홍보=표시광고법·개인정보 동의.
- DB: public 스키마(profiles_personal/profiles_company, job_posts, job_applications, site_promotions). tenant 격리 아님.

## E-2. 전자 해촉증명서 (프리랜서 편의 — 차별 기능)
- **발급주체(시행/대행 법인) 등록 + 직인 이미지 등록**(Supabase Storage 보유). 법인만 발급권한.
- 발급주체: 현장 근무·퇴사 프리랜서를 **개별/일괄 선택** → 증명서 자동채움(조직도 멤버십 이력=근무기간 + 수수료/원천징수 `sales_withholding_statements`=수수료 수령자·소득) → **등록 직인 날인** → 발송.
- 프리랜서: 연말 **본인 근무이력(현장·기간) 자동표시** → 현장 선택 → **발급신청(개별/일괄)** → 관리페이지 **기간별·현장별 관리** → **일괄 출력(인쇄·PDF)**.
- 구현자산: 조직도 멤버십이력·수수료/원천징수·이미지업로드(직인)·reportlab(한글PDF)·일괄처리 전부 보유.
- ★차별: 발급증명서를 **해시체인 원장 기록**→위변조 불가(법인 부인불가·프리랜서 위조불가), 검증QR/해시 진위확인.
- 정직: 법정통일양식 아님(실무항목 포함, 3.3% 세무신고용), 전자직인=전자문서법·무결성은 해시체인 보강, 주민번호 마스킹/암호화·발급동의.
- DB: cert_issuers(직인), termination_certificates(발급), cert_requests(신청). 발급주체=법인(tenant), 프리랜서 신청=개인(공통 연계).

## F. 소셜·네트워크 (lock-in)
- **친구추가(소셜그래프)** + **단톡(그룹채팅, WebSocket 보유)** + 앱푸시(FCM 보유) + 다중톡(알림톡 bulk).
- **승진→네트워크 모집**: 직원→팀장→본부장 승진 시 친구등록 동료에게 모집공고 홍보. 인맥기반 구인 바이럴.
- 단톡 메시지영속·읽음·미디어 추가구축. 광고성정보 동의관리.

## F-2. 모델하우스 데스크 (대부분 보유 + 동의팝업 보강)
- 보유: mh_desks/visitors/consents/staff_match/notifications/visit_stats → 방문고객 등록·지정방문자·직원검색/매칭·공통알림.
- ★보강: **방문객 개인정보 동의 팝업**(수집항목·목적·보유기간 고지+동의저장 mh_consents, 마케팅 별도동의). 데스크 직원 현장로그인→방문관리→등록시 동의팝업→직원매칭→알림.

## F-3. ★수수료 더치페이 (합의기반 분배 — 신뢰 차별점)
- 보유: 수수료 9테이블(splits/approvals/settlements...) + 배분검증 + 무결성가드(배분초과 적발).
- 협업분배: 직원↔직원/팀장↔직원/본부장↔직원이 한 계약 수수료를 **비율 또는 금액**으로 분배(카카오뱅크 더치페이형).
- **사전 다자동의**: 참여자 전원 사전동의해야 확정(전자서명/동의기록). **변경=전원 재동의** 없으면 불가(일방변경 차단).
- **해시체인 무결성**: 합의·변경 이력 원장기록→위변조불가·분쟁증거(스마트계약형). 분양 수수료 분쟁 구조적 예방=독보적 신뢰.

## G. 11모듈(보유) + 시행사 통합 모니터링
세대배치도(실시간 동호선점 Redis락+TTL+DB UNIQUE)·분양가(AVM연동)·청약당첨(추첨/가점엔진 보강)·수납(가상계좌 헥토 webhook 자동대사-Phase2)·중도금대출(LTV/DSR엔진-Phase2)·전매신고·세금보증(세법엔진통합-Phase3)·조직도·수수료·방문데스크·AI예측/무결성가드(해시체인 결합). 시행사 /projection/summary 실시간 push화.

## H. 디자인
- ui-ux-pro-max/designer 디자인스킬 적용: 세련·가독·직관 극대화. 분양 현장앱 디자인시스템+핵심화면(현장리스트·세대배치도·고객·직원·대시보드). 모바일 우선(설치형 PWA).

## 단계
- **Phase 0(선결·보안)**: RLS ENABLE(카나리→전체, app bypassrls라 안전)·인덱스·이중방어. 88테이블 이미 존재.
- **Phase 1(분양앱 핵심)**: A~F + 동호선점 동시성 + 서브도메인배선 + 실시간 push + 디자인.
- **Phase 2(자금실연동)**: 가상계좌 webhook·LTV/DSR·청약홈 OpenAPI.
- **Phase 3(고도화)**: 세법엔진통합·HUG·AI분양속도예측·무결성 해시체인 외부앵커링.
