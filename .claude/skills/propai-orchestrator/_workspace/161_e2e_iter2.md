## 잔여 오류 (심각도·근본원인·수정위치)

### [P0-Critical] 설계 용도지역 미상속 → 용적률 환각 (정합/할루시네이션)
- 증상: 일반상업지역(조례 800%) 프로젝트인데 설계 단계가 "제2종일반주거지역/250%"로 계산
- 근본원인: 설계 페이지가 프로젝트 컨텍스트의 zone_type(일반상업)을 상속하지 않고 building_type=공동주택→주거 기본 zone 폴백. (메모리 기존 메모와 일치: "자동산출 폴백 zone_code 매핑 한글 용도지역명 부정확")
- 수정위치(추정): 프론트 설계 페이지 `apps/web/.../projects/[id]/design`의 초기 zone 주입 로직 + 컨텍스트 useProjectContextStore의 zone_type 바인딩
- 파급: 설계 연면적 과소 → 수지 ROI -72%·등급E 환각 → 보고서 왜곡

### [P0-Critical] 부지 심층분석 크래시 (site-analysis/analyze 404 + toLocaleString)
- 증상: "심층 분석" 클릭 시 에러바운더리. 페이지 전체 다운.
- 근본원인: 프론트가 POST /api/v1/site-analysis/analyze 호출하나 백엔드 라우트 부재(404). 응답 undefined의 .toLocaleString() 호출이 렌더 크래시.
- 수정위치: (a) 백엔드 site-analysis/analyze 라우트 추가 또는 프론트가 실제 존재 엔드포인트(zoning/analyze+avm+nearby-map 조합) 사용하도록 정정, (b) 프론트 숫자 렌더에 옵셔널 체이닝/널가드(`?.toLocaleString()`)

### [P1-High] 인허가 UI 용도지역 미전파 + AI진단 무한 분석중
- 증상: 매트릭스 "용도지역 정보가 없습니다", AI 진단 영구 "분석 중"
- 근본원인: 백엔드 ai-analysis는 200 정상이나, 프론트 인허가 페이지가 프로젝트 zone_type을 매트릭스/진단 입력으로 전달하지 않음 + 진단 응답 바인딩/await 미완료(설계와 동일 SSOT 누수)
- 수정위치: 프론트 인허가 페이지 `projects/[id]/permit` — 컨텍스트 zone_type 주입 + ai-analysis 응답 상태 핸들링(로딩 해제)

### [P2-Medium] 전역 toLocaleString 콘솔에러 (공용 컴포넌트)
- 증상: 다수 페이지에서 동일 스택의 비치명 콘솔에러 반복(레이아웃/헤더 추정, 코인잔액 등 숫자 포맷)

### [P2-Medium] 단계완료 영속 누락 (법규검토 등)
